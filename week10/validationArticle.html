Validating Forms

  1. HTML Validation Form Example
  2. JavaScript Validation Form Example
  • Client-side form validation ensures that the data submitted to the server matches the requirements set forth in the controls.
  • Client-side validation should not be considered an exhaustive security measure. The apps should always perform checks on any form-submitted data on the server-side as will.
  • Form validation is when the form provides a message to the user that the information is incorrect or incomplete.
  • The form submission should only occur if the information is correctly formatted.
  • Built-in form validation uses HTML5 validation features.
  • JavaScript form validation is coded using JavaScript and is completely customizable.
  • The HTML5 form validation features are:
    • required: the form must be filled in.
    • minlength and maxlength: specify min and max length of text strings.
    • min and max: specify min and max of numerical values.
    • type: specify data type
    • pattern: specify a regexp that defines a pattern for the entered data to follow.
  • Valid html elements gain the :valid CSS pseudo-class
  • Invalid html elements gain the :invalid CSS pseudo-class.
  • It is required by the WCAG accessibility guidelines to indicate which form fields are required. You should only require users to input data that is actually needed.
  • Regular expressions (regexps) are a pattern that can be used to match character combinations in text strings.
  • Regexps are case-sensitive, but you can use make them accept both capitalized and lowercase using an [Aa] pattern.
  • Instead of using a regexp, you can sometimes specify a specific type, such as the email type, to indicate the formatting pattern.
  • Number inputs can include number ranges using min and max, and also a step attribute for incrementing by certain amounts.
  • There are different ways to use JavaScript for validation.
  • The Constraint Validation API provides form element DOM interfaces and properties for them.
  • The constraint validation API can allow customization of error messages.
